# Requisitos No Funcionales

Los siguientes requisitos no funcionales aseguran la calidad, seguridad y usabilidad del sistema.

---

## [P0] Críticos

1. El sistema debe garantizar una disponibilidad del 99% para no interrumpir el seguimiento de medicamentos.  
2. Toda la información personal y médica debe estar cifrada en tránsito y en almacenamiento local.  
3. El sistema debe emitir notificaciones puntuales con respecto a la hora programada.

---

## [P1] Importantes

4. La interfaz debe ser accesible y usable para adultos mayores, con fuentes legibles y navegación intuitiva.  
5. El sistema debe estar optimizado para funcionar en dispositivos con bajos recursos.  
6. La aplicación debe mostrar un mensaje de confirmación al realizar acciones críticas como eliminar medicamentos. 
7. El sistema debe permitir exportar la información del usuario en formato PDF y JSON.  

---

## [P2] Deseables

8. La app debe mostrar consejos o tutoriales breves para nuevos usuarios.

9.La aplicación debe permitir cambiar el tamaño del texto en toda la interfaz.

10.La interfaz debe mantener el mismo diseño y comportamiento en dispositivos de diferentes marcas.
